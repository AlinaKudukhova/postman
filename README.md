# Коллекция Postman: Работа с REST API для сайта CakeEasy с помощью DevTools

Коллекции запросов в Postman для тестирования сценариев работы с корзиной и оплатой на сайте [CakeEasy](http://dev.caketeasy.se/). 
Запросы были скопированы через DevTools и импортированы в Postman.

## Файл коллекции: [Cake](Cake.postman_collection.json)

### Сценарий 1: Добавление товаров в корзину и проверка содержимого

### Описание
Создана коллекция запросов для тестирования сценария добавления товаров в корзину. Основная задача — проверить интеграцию между модулями системы, не занимаясь отладкой дефектов.

### Основные шаги
1. **Регистрация нового пользователя** через `POST /auth/register/`.
2. **Авторизация пользователя** через `POST /auth/login/`.
3. **Получение списка продуктов** по категории через `GET /api/store/product-types/`.
4. **Добавление товара в корзину** через `POST /store/cart/order-lines/`.
5. **Проверка содержимого корзины** через `GET /store/cart/`.
6. **Удаление товаров из корзины** через `DELETE /store/cart/order-lines/`.

### Использованные технологии
- **Postman** — для создания и автоматизации коллекции.
- **DevTools** — для копирования запросов.
- **REST API** — для взаимодействия с сервером.

### Сценарий 2: Оформление заказа и оплата

### Описание
Создана коллекция запросов для тестирования полного цикла оформления заказа и оплаты. Сценарий включает регистрацию, авторизацию, добавление товаров в корзину, выбор доставки и оплату картой.

### Основные шаги
1. **Регистрация и авторизация пользователя**.
2. **Получение списка продуктов** и добавление товара в корзину.
3. **Выбор доставки** и оформление заказа через `POST /store/cart/delivery-order/`.
4. **Оплата заказа** через `POST /store/payments/bambora/`.
5. **Проверка статуса оплаты** через `GET /store/payments/{{sid}}/status/`.
6. **Подтверждение оплаты** и проверка заказа через `GET /store/orders/{{sid}}/`.

### Использованные технологии
- **Postman** — для создания коллекции.
- **DevTools** — для копирования запросов.
- **REST API** — для взаимодействия с сервером.
- **Интеграция с платежной системой Bambora** — для тестирования оплаты.

### Общее описание

Оба сценария направлены на тестирование бэкенда сайта CakeEasy с использованием Postman и DevTools. Основная цель — проверка корректности работы API и интеграции между модулями системы. В процессе тестирования были выявлены дефекты, связанные с несоответствием фронта и бэка.

# Коллекция Postman: Работа с API NASA Mars Photos

Автоматизированная коллекция запросов для тестирования и взаимодействия с API [NASA Mars Photos](https://api.nasa.gov/).

## Файл коллекции: [NASA](NASA.postman_collection.json)

### Описание

Выполнен GET-запрос к API NASA Mars Photos для получения фотографий, сделанных марсоходом Curiosity на указанную дату (`earth_date=2015-06-03`). В Postman были написаны автоматизированные тесты для проверки корректности ответа сервера.

### Основные проверки

- Статус ответа (200 OK).
- Тип данных ответа (массив фотографий).
- Совпадение даты из запроса с датой в ответе.
- Проверка типов данных для каждого поля в объектах фотографий, камеры и марсохода.
- Наличие обязательных полей в объектах фотографий, камеры и марсохода.

# Коллекция Postman: Работа с REST API для Vikunja

Коллекция запросов в Postman для тестирования API Vikunja. Основная задача — проверить работу различных эндпоинтов, включая регистрацию пользователей, авторизацию, создание и управление метками (labels). В коллекции реализованы как **позитивные**, так и **негативные** сценарии, а также проверки для получения определенных статус-кодов.

## Файл коллекции: [Vikunja Practice](<Vikunja Practice.postman_collection.json>)

## Основные сценарии

### 1. Работа с метками (Labels)
- **Позитивные проверки**:
  - Создание метки с корректными данными (статус **201**).
  - Получение информации о метке (статус **200**).
  - Удаление метки (статус **200**).

- **Негативные проверки**:
  - Создание метки с некорректными данными (null, число, boolean, пустое значение) — статус **400**.
  - Создание метки с длиной `title` более 250 символов — статус **400**.
  - Попытка создания метки без авторизации — статус **401**.
  - Попытка доступа к несуществующей метке — статус **404**.
  - Попытка создания метки с неверным Content-Type — статус **415**.

### 2. Авторизация (Login)
- **Позитивные проверки**:
  - Успешная авторизация с корректными данными (статус **200**).
  - Проверка возврата JWT-токена и его валидности.

- **Негативные проверки**:
  - Авторизация с неверным паролем — статус **412**.
  - Авторизация с несуществующим пользователем — статус **404**.

### 3. Регистрация пользователя (Register)
- **Позитивные проверки**:
  - Успешная регистрация нового пользователя (статус **200**).
  - Проверка структуры ответа (наличие полей `id`, `name`, `username`, `email`, `created`, `updated`).

- **Негативные проверки**:
  - Регистрация с некорректными данными (например, пустой email или username) — статус **400**.
